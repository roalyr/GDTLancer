## CURRENT GOAL: Discovered Station Market Seeding

- TARGET_SCOPE: When a procedural sector is discovered and its station entry is created in `GameState.locations`, that station dict must include a seeded `market_inventory` field in the canonical `{commodity_template_id: {buy_price: int, sell_price: int, quantity: int}}` shape so that: (a) the player docked-trade UI renders a real market at discovered stations instead of an empty panel, and (b) NPC dock-trade buy/sell paths that already guard on `market_inventory.empty()` now find non-empty inventories and execute real quantitative trades. Market seeding must be deterministic from the world seed and sector id so that save/load round-trips reproduce the same authored baseline. Seeded discovered station inventory must persist through save and reload the same way authored `.tres` location inventories already do via `GameStateManager._serialize_resource_dict_by_string_key`. This milestone does not introduce per-sector price curves, dynamic restocking, supply/demand balancing, or faction-owned commodity lists — the seed is a static baseline that only changes through runtime player/NPC buy/sell mutations.
- TARGET_FILES:
  - src/core/simulation/agent_layer.gd — owns `_generate_procedural_station_for_sector(...)`, the function that writes the plain dict into `GameState.locations[station_id]`; extend that dict to include a seeded `market_inventory` field using a deterministic RNG keyed on world seed + station id.
  - src/tests/core/simulation/test_agent_layer.gd — add focused deterministic coverage for: discovered station dict contains a non-empty `market_inventory` after generation, NPC dock-trade sell/buy executes against the seeded discovered station inventory (quantity changes), and the seeded inventory shape matches `{commodity_template_id: {buy_price: int, sell_price: int, quantity: int}}`.
  - src/tests/autoload/test_game_state_manager.gd — add or extend save/load round-trip coverage to confirm a plain-dict discovered location entry (with a seeded `market_inventory`) survives a save/load cycle with its mutated quantity intact.
